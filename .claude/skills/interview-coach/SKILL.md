# 面试模拟教练 — Interview Coach

## 功能概述

本 Skill 用多个 AI Agent 并行分析你的简历和岗位 JD，从不同面试官视角生成问题 + 参考答案：

1. 解析简历，提取技能树、项目经历、教育背景、实习经历
2. 解析 JD（可选），提取岗位核心要求、技术栈、加分项
3. 启动 3 个 Agent 模拟不同角色面试官，并行出题
4. 汇总去重，生成完整面试准备文档
5. 输出到桌面 `面试准备_YYYYMMDD_HHMM.md`

---

## 执行流程

### Step 1：收集输入

若用户未提供，主动询问：

- **简历**：文件路径 (.docx/.pdf/.txt)，或直接粘贴文本
- **JD**（可选）：文件路径，或直接粘贴文本，或先跳过只用简历出题
- **目标岗位**（可选）：公司/岗位名，用于针对性出题
- **输出路径**：默认桌面

### Step 2：读取文件内容

若用户提供文件路径，调用 read_resume.py 解析：

```bash
python scripts/read_resume.py --resume <简历路径> --jd <JD路径可选>
```

脚本输出简历原文和 JD 原文到终端，同时缓存到 `.interview_cache.json`。

若用户直接粘贴文本，跳过此步。

### Step 3：并行 Agent 分析

同时启动 3 个 Agent，分别从不同角度分析：

**Agent 1 — 简历挖掘者**
```
你是简历分析专家。分析以下简历内容，输出 JSON：

{
  "skills": ["技能1", "技能2", ...],
  "projects": [{"name": "项目名", "role": "角色", "tech": ["技术"], "highlight": "亮点"}],
  "experience": [{"company": "公司", "role": "岗位", "duration": "时间", "achievements": ["成果"]}],
  "education": {"school": "学校", "major": "专业", "degree": "学位"},
  "weak_points": ["简历中的薄弱点1", ...],
  "interview_hooks": ["面试官可能会追问的点1", ...]
}

简历内容：
<在此插入简历文本>
```

**Agent 2 — JD 分析师**（若用户提供了 JD）
```
你是岗位分析专家。分析以下 JD，输出 JSON：

{
  "must_have": ["必须掌握的技能/经验"],
  "nice_to_have": ["加分项"],
  "core_responsibilities": ["核心职责"],
  "tech_stack": ["技术栈"],
  "soft_skills": ["软技能要求"],
  "keywords": ["JD高频关键词"],
  "hidden_requirements": ["隐含要求推测"]
}

JD 内容：
<在此插入 JD 文本>
```

**Agent 3 — 面试题库构建者**
```
你是有 10 年经验的技术面试官。根据以下信息，生成面试问题 + 参考答案。

简历分析结果：<简历分析JSON>
JD 分析结果：<JD分析JSON，如无则标注"未提供">

要求：
1. 生成 8-12 个面试问题
2. 覆盖以下类别：
   - 自我介绍类（1-2题）
   - 技术能力类（2-3题，针对简历中的技能栈）
   - 项目经验类（2-3题，深挖简历中的项目细节）
   - 行为面试类（1-2题，STAR法则场景题）
   - JD 针对性类（2-3题，JD 要求的技能/场景，如无 JD 则替换为综合素质题）
3. 每个问题附带参考答案（基于简历真实内容，不虚构）
4. 每个问题标注难度（⭐~⭐⭐⭐⭐⭐）和考察意图
5. 对于技术题，给出回答要点而非完整答案
6. 对于项目题，给出 STAR 框架建议
```

### Step 4：汇总与润色

将 3 个 Agent 的结果汇总：

1. 去重：删除语义重复的问题
2. 排序：按面试自然流程排列（自我介绍 → 技术 → 项目 → 行为 → JD/综合）
3. 补充：为每个问题添加"面试官追问"子问题（1-2个）
4. 润色：确保答案口语化、可直接使用

### Step 5：输出文档

生成 Markdown 格式面试准备文档，保存到桌面：

```
📄 面试准备文档 — [岗位名/通用]
👤 姓名：[从简历提取]
📅 生成时间：[时间戳]

---

## 一、简历速览

[简历关键信息摘要：学历、技能、核心经历]

## 二、JD 核心要求（如有）

[JD 关键词提取 + 简历匹配度速评]

## 三、面试问题与参考答案

### Q1: [问题内容]
- 🏷️ 类别：技术能力 / 项目经验 / 行为面试 / ...
- 📊 难度：⭐⭐⭐
- 🎯 考察意图：[一句话说明面试官想了解什么]
- 💬 参考回答：[基于简历真实经历的回答]

> ⚡ 可能的追问：[1-2个潜在追问 + 简短应对建议]

### Q2: ...

---

## 四、面试技巧速查

- 针对这份简历的 3 个加分点
- 需要提前准备的 3 个薄弱问题
- JD 匹配度最高的 3 个关键词（面试中主动提及）
```

用 Write 工具将内容写入 `桌面/面试准备_YYYYMMDD_HHMM.md`。

### Step 6：确认交付

告知用户输出路径，展示问题概要，询问是否需要调整或深入某个问题。

---

## 注意事项

- **不虚构内容**：所有回答基于简历真实信息，不编造经历或数据
- **保持真实**：若简历某项技能薄弱，答案中如实反映而非过度美化
- **JD 驱动**：有 JD 时优先对标 JD 出题；无 JD 时聚焦简历通用面试题
- **口语化**：参考答案用口语化中文，方便用户直接练习
- **追问机制**：每个问题附带追问，模拟真实面试压力

---

## Agent 使用说明

本 Skill 的核心是并行 Agent 分析。执行 Step 3 时，在一个消息中同时发出 3 个 Agent 调用
（Agent 1 和 Agent 2 可并行，Agent 3 依赖前两者结果所以等它们完成后再发）。

缩减版流程（JD 未提供时）：Agent 1 和 Agent 3 并行，Agent 3 直接基于简历出题。
